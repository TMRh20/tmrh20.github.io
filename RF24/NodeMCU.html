NodeMCU v1.0

RF24 is fully compatible with ESP8266 boards
First, ensure being using the latest hardware support version into Arduino IDE as old version have issues. A lot of ressources on interner explain how to proceed, but using https//github.com/esp8266/Arduino#using-git-version-basic-instructions (git version) is strongly advised.


Common issues

RF24 on ESP8266 based board have common issues. The most important one seems to be the wrong SPI use. On most ESP8266 like the NodeMCU v1.0 one, there are two availaible SPIs ports. The one to use here is the HSPI, as described in this pinout https://4.bp.blogspot.com/-CsD9T4NUeMw/VrjblpNQzZI/AAAAAAAAAQs/yMUWFV1qsk4/s1600/NodeMCU_Pinout.png on GPIO 12-14.

Another important issue is the lack of power. It is advised to put some external capacitors for power rail decoupling ( < 22nF ) and another one as power reserve ( > 22Âµf ).

Some low cost RF24 modules are not working with the AutoACK functionnality enabled. You can try disabling it with adding 

 radio.setAutoAck(false);
into the setup() function, after the begin() call.


Debug

For debugging purposes, you could try a call to 

printDetails();
to ensure the communication with the RF24 module is working properly.
